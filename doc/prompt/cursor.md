# 🧠 一、整体工程流（你要建立这个脑图）

```text
需求（你的文档）
   ↓
拆任务（你已经有了 ✅）
   ↓
Cursor 单任务生成代码
   ↓
人工 Review（关键！）
   ↓
Git 提交（原子提交）
   ↓
继续下一个任务
```


---

# 🔥 二、核心原则（非常重要）

### 1️⃣ 永远“小步提交”（禁止一把梭）

❌ 错误：

```bash
feat: 完成整个RAG系统
```

✅ 正确：

```bash
feat: 初始化前端项目
feat: 实现文件上传组件
feat: 实现embedding服务
```

---

### 2️⃣ 一个任务 = 一个 commit

👉 你的文档刚好已经满足这个原则（非常优秀）

---

### 3️⃣ Cursor 只负责“写代码”，你负责“做决策”

👉 你是 Tech Lead，不是打字员
