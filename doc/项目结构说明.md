# 适配本项目的极简 Monorepo 目录结构
遵循 “轻量级、结构清晰、贴合分层约束” 的原则，设计如下目录（无重型工具，仅用 npm/yarn workspaces）：
```plaintext
ai-knowledge-assistant/  # 根目录（monorepo 根）
├── package.json         # 根配置：声明 workspaces、统一脚本（如 dev/start）
├── tsconfig.json        # 全局 TS 配置（前后端共享基础配置）
├── .env                 # 全局环境变量（如豆包 API 密钥，后端读取）
├── packages/            # 子包目录
│   ├── frontend/        # 前端子包（Vue3 + TS + Vite）
│   │   ├── src/         # 前端源码（按 views/components/utils/api 分层）
│   │   ├── package.json # 前端依赖（vue、vite、axios、pinia 等）
│   │   └── vite.config.ts # 前端构建配置
│   ├── backend/         # 后端子包（Express + TS）
│   │   ├── src/         # 后端源码（按 routes/services/utils/types 分层）
│   │   ├── package.json # 后端依赖（express、multer、pdf-parse 等）
│   │   └── tsconfig.json # 后端 TS 配置（继承根配置）
│   └── shared/          # 共享子包（仅放通用类型/工具）
│       ├── src/
│       │   ├── types/   # 前后端共用 TS 类型（ChatItem、HistoryItem、流式响应等）
│       │   └── utils/   # 通用工具（如字符串处理、类型校验）
│       └── package.json # 共享依赖（仅基础 ts 依赖）
└── README.md            # 项目说明
```
